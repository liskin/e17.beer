# AGENTS.md - AI Agent Guide for e17.beer

Quick reference for AI agents working on this repository.

## What This Is

A Jekyll-based website (<https://e17.beer/>) for E17 (Walthamstow, London) running clubs, breweries, and local venues.
The unique feature is an interactive opening hours visualization for the Blackhorse Beer Mile.

## Project Layout

- **Root** = Jekyll website project
  - `_config.yml`, `Gemfile` = Jekyll config and Ruby deps
  - `Justfile` = Build commands (see below)
  - `_data/places.json` = **SYMLINK** to `_scripts/_data/places.json` (generated venue data)
  - `_data/*.yaml` = Editable data files (links, bio, social)
  - `hours/` = Custom visualization page (standalone HTML/CSS/JS)

- **`_scripts/`** = Python project for data generation
  - `pyproject.toml` = Python deps and config (uses `uv`)
  - `Justfile` = Script commands (see below)
  - `_data/venue_metadata.json` = Intermediate data (place names, IDs, happy hours from spreadsheet)
  - `_data/places.json` = Final generated venue data (opening hours, GPS coordinates, etc.)
  - `fetch_venue_metadata.py` = Fetches spreadsheet → generates `venue_metadata.json`
  - `update_places.py` = Fetches Google Places data → generates `places.json`
  - Unit tests mock Google API responses

## Build Commands

### Jekyll Website (from root)
```bash
just build        # Build to _site/
just serve        # Serve at http://localhost:12345 (note: non-standard port!)
```

### Python Scripts (from _scripts/)
```bash
just venv         # Setup venv with uv
just test         # Run pytest
just fmt          # Format with ruff
```

**⚠️ DON'T RUN**: `just update-places` or `just fetch-venue-metadata` need `GOOGLE_MAPS_API_KEY`. Use unit tests instead.

## Non-Obvious Details

### Data Generation
- `_data/places.json` is a **symlink** to `_scripts/_data/places.json` (generated by Python scripts)
- Never edit `places.json` or `venue_metadata.json` manually
  (if you need to change their structure, use Python to load/manipulate/save the JSON)
- `places.json` contains `keyframe_periods` (% of week) used for CSS animation of opening hours
- Happy hours include HTML `<span>` tags with classes for styling
- Other YAML files in `_data/` can be edited directly
- Data pipeline: spreadsheet → `fetch_venue_metadata.py` → `venue_metadata.json` → `update_places.py` → `places.json`

### Opening Hours Visualization
- Located in `hours/index.html` with CSS/JS in `hours/_includes/`
- Uses CSS `@keyframes` with `--open` custom properties
- Keyframe percentages represent week progress (Sunday 00:00 = 0%, Saturday 23:59 ≈ 100%)
- See `get_week_percentage()` function in `update_places.py` for calculation logic

### Jekyll Theme
- Uses `linkhub-jekyll-theme` - most UI comes from the theme
- `index.md` is minimal - just front matter
- Custom pages like `/hours/` are standalone

### Python Testing
- All scripts have unit tests (with mocked API responses where necessary)
- Tests are the way to validate changes without API keys

## Quick Tips

1. **Build system**: Use `just` commands, not direct tool invocation. The `Makefile` just redirects to `just`.
2. **Port number**: Jekyll serves on 12345, not the default 4000
3. **API key**: Don't try to run data generation scripts without `GOOGLE_MAPS_API_KEY`
4. **Testing**: Always use unit tests for Python changes
5. **Dependencies**: Check `pyproject.toml` for Python version/deps, `Gemfile.lock` for Ruby

## Where to Find Details

- Python version and dependencies: `_scripts/pyproject.toml`
- Jekyll dependencies: `Gemfile` and `Gemfile.lock`
- Build/test commands: `Justfile` in root and `_scripts/`
- Data structure: Look at `_scripts/_data/places.json` and unit tests
- Code conventions: Follow existing patterns, ruff formatting for Python
