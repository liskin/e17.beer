## Repository Overview

**e17.beer** is a Jekyll-based website for E17 (Walthamstow, London) running clubs, breweries, happy hours, and other local information. It features:
- A main landing page with links to calendars, beer mile info, and local discounts
- An interactive opening hours visualization for the Blackhorse Beer Mile venues
- Python scripts that fetch and process venue data from Google Places API

**Website**: https://e17.beer

## Build System

The project uses **Just** (a modern command runner) for task automation. There's a `Makefile` that redirects to `just` for backwards compatibility.

### Root Justfile (Jekyll website)

Located at: `/home/runner/work/e17.beer/e17.beer/Justfile`

Key commands:
- `just build` - Build the static website into `_site/` directory
- `just serve` - Serve the site with livereload at http://localhost:12345
- `just serve-public` - Serve publicly at 0.0.0.0:12123
- `just clean` - Clean build artifacts
- `just bundle-install` - Install Ruby dependencies
- `just bundle-update` - Update Ruby dependencies

The site uses Jekyll 4.4.0 with the `linkhub-jekyll-theme` (v0.1.6).

### Scripts Justfile (Python tools)

Located at: `/home/runner/work/e17.beer/e17.beer/_scripts/Justfile`

Key commands:
- `just venv` - Create Python virtual environment using `uv sync --exact`
- `just test` - Run pytest tests
- `just fmt` - Format code with ruff (format + import sorting)
- `just update-places` - Update venue data from Google Places API (requires API key)
- `just generate-place-ids` - Generate place IDs (requires API key)

**Important**: The `update-places` and `generate-place-ids` commands require a `GOOGLE_MAPS_API_KEY` environment variable. Don't try to run these without the key - use unit tests instead to validate changes.

### Main Scripts

#### `update_places.py`
Updates venue information from Google Places API and generates `_data/places.json`.

Features:
- Fetches place data including opening hours
- Formats happy hours with HTML spans for styling
- Calculates keyframe percentages for opening hours animation
- Has comprehensive unit tests in `test_update_places.py`

#### `generate_place_ids.py`
Generates Google Place IDs for venues by searching the API.

Features:
- Searches for venues in the E17 area
- Has special overrides for accuracy (e.g., "Hackney Church" → "Hackney Church Blackhorse")
- Uses location biasing (center: 51.5886, -0.0118, radius: 5000m)
- Has unit tests in `test_generate_place_ids.py`

#### `utils.py`
Shared utilities:
- Custom logging formatters with emojis
- Context manager for logging context
- CLI verbosity options
- Google Places client initialization

### Testing
All scripts have unit tests that mock the Google API responses. Run tests with:
```bash
cd _scripts
just test
```

Tests are the best way to validate changes without needing an API key.

## Opening Hours Visualization

The `/hours/` page shows an interactive visualization of Blackhorse Beer Mile opening hours.

**How it works**:
1. Jekyll reads venue data from `_data/places.json`
2. For each venue, generates CSS `@keyframes` animations based on `keyframe_periods`
3. The keyframes set `--open: 1` or `--open: 0` at specific percentage points in the week
4. JavaScript and CSS use these values to show which venues are currently open
5. Includes a "time machine" feature to view hours at different times

**Key fields in `places.json`**:
- `place_name` - Venue name
- `place_id` - Google Place ID
- `url` - Google Maps URL
- `location` - {lat, lng}
- `happy_hours` - Array of happy hour offers (with HTML spans for styling)
- `keyframe_periods` - Array of {open, close} percentages for animation

The `keyframe_periods` represent percentages of a week (Sunday 00:00 = 0%, Saturday 23:59 = ~100%).

## Development Workflow

### Making Code Changes

1. **Python scripts changes**:
   - Navigate to `_scripts/`
   - Make changes to Python files
   - Run `just fmt` to format code
   - Run `just test` to run tests
   - Add/update unit tests as needed
   - DO NOT run `update-places` or `generate-place-ids` without API key

2. **Website changes**:
   - Edit Jekyll files (HTML, CSS, JS, YAML, MD)
   - Run `just build` to build the site
   - Run `just serve` to test locally at http://localhost:12345
   - Check the `_site/` directory for generated output

3. **Data changes**:
   - `_data/places.json` is generated - don't edit manually
   - `_data/links.yaml`, `_data/bio.yaml`, `_data/social.yaml` can be edited directly
   - After editing YAML files, rebuild the site

### Git Workflow

- Current branch: `claude/add-agents-md-documentation`
- Use `report_progress` to commit and push changes (don't use `git push` directly)
- Clean working directory at start

## Common Tasks

### Add a new venue to the data
- Don't do this without API key - would need to modify `update_places.py` input data
- Use unit tests to validate format changes

- Edit `format_happy_hours_line()` in `update_places.py`

- Edit `hours/index.html` for HTML structure

- Ruby: `just bundle-update`

## Important Notes

### For AI Agents

1. **Don't guess API responses**: Use existing test data and unit tests instead of trying to call the Google Places API
2. **Use unit tests extensively**: All Python changes should be validated with tests
3. **Respect the build system**: Always use `just` commands, not direct tool invocation
4. **The Makefile redirects to just**: Use `just` directly instead
5. **Jekyll site structure**:
   - The main content is minimal (index.md is nearly empty)
   - Most UI comes from the `linkhub-jekyll-theme`
   - Custom pages like `/hours/` are standalone
6. **Data is generated**: Don't manually edit `places.json` - it's generated by Python scripts
7. **Testing locally**:
   - Use `just serve` for the website
   - Use `just test` for Python scripts
   - The site runs on port 12345 (not the default Jekyll port)

### Key Conventions

- **Python**: Follow ruff formatting, use type hints, write unit tests
- **Keyframe calculations**: The `get_week_percentage()` function is critical for the hours visualization
- **Happy hours HTML**: Uses `<span>` tags with classes `offer-name` and `offer-time` for styling
- **Location biasing**: E17 area centered at (51.5886, -0.0118) with 5km radius

### Limitations

- Google Places API requires authentication - can't be tested without API key
- The site is static (Jekyll) - no server-side processing
- Opening hours visualization assumes venues aren't open 24 hours (see TODO in code)
